# MeatyPrompts Design System

Welcome to the MeatyPrompts design system - a comprehensive collection of reusable components, design tokens, and patterns built for creating consistent, accessible user interfaces.

## üéØ Mission

Our design system makes world-class prompt-crafting as simple, repeatable, and measurable as spell-check. We provide the building blocks for creating interfaces that are:

- **Consistent** across all MeatyPrompts applications
- **Accessible** to users of all abilities (WCAG 2.1 AA)
- **Scalable** for rapid product development
- **Maintainable** with centralized design decisions

### Complications & Adapter System
The `PromptCard` component exposes named slots where small widgets‚Äîcalled
**complications**‚Äîcan render additional context. Each complication is provided
through an adapter that receives card state and returns a React element. This
adapter pattern lets applications extend cards with metrics, status indicators,
or custom plugins without changing the core component.

```tsx
import { PromptCard } from '@meaty/ui';
import { MetricComplication } from './complications/examples/MetricComplication';

<PromptCard
  title="My Prompt"
  complications={{
    bottomRight: { component: MetricComplication, props: { value: 42, unit: '%' } },
  }}
/>;
```

## üé® Design Principles

### Consumer Polish
Inspired by Fluent 2 and Adobe Spectrum, our components feature rounded corners, subtle shadows, and friendly micro-interactions that feel polished and approachable.

### PatternFly Density
We adopt PatternFly's thoughtful approach to information density, particularly in dashboards and data-heavy interfaces, while maintaining visual clarity.

### Radix Foundation
Built on Radix primitives for robust accessibility, keyboard navigation, and screen reader support that meets WAI-ARIA authoring practices.

## ‚ôø Accessibility First

Every component is built with accessibility as a core requirement:

- **Keyboard Navigation** - Full keyboard operability
- **Screen Readers** - Proper ARIA attributes and semantic HTML
- **Focus Management** - Visible focus indicators and logical tab order
- **Color Contrast** - WCAG AA contrast ratios across all themes
- **Motion Sensitivity** - Respects `prefers-reduced-motion`
